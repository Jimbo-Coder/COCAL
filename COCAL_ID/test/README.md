# COCAL_ID tests

These tests use reduced COCAL-format initial data generated by sampling larger
COCAL outputs.  The fixtures are meant to exercise the COCAL_ID file readers,
grid setup, interpolation, HydroBase/ADMBase population, and cleanup paths; they
are not converged or physically meaningful initial data.

Directory layout:

- `test_InitialData/`: reduced COCAL-format input data read by the parfiles.
- `cocal_*.par`: testsuite parameter files.
- `cocal_*`: checked-in scalar reduction outputs used as test references.  The
  Cactus testsuite requires each `cocal_*.par` file to have a same-named
  expected-output directory at this level.

The scalar references use `IOScalar::one_file_per_group = "yes"` and include
the full ADM lapse, shift, metric, and curvature groups, plus HydroBase fluid
and magnetic/vector-potential groups where applicable.

The default reduced fixture size is:

- gravitational grid: `nrg=26`, `ntg=4`, `npg=4`
- fluid grid: `nrf=8`, `ntf=4`, `npf=4`
- Cactus grid: one Carpet level on `[-1,1]^3` with `dx=dy=dz=0.5`

`nrg=26` is the smallest gravitational radial grid found to pass the BNS grid
construction paths, which hard-code enough inner radial structure to fail at
`nrg=25`.  `nrf=8` is the smallest fluid radial grid found to pass the RNS and
MRNS radial coordinate setup while preserving source-like radial ratios.
`nt=np=4` keeps a non-degenerate five-point angular grid and preserves the
quarter-turn indexing assumptions used by the COCAL readers.

Regenerate the fixtures with:

```sh
COCAL_ID/utils/reduce_cocal_fixture.py
```

By default the script looks for source data in a sibling `COCAL_IDs` directory
next to the ETK checkout.  Use `--root /path/to/COCAL_IDs` to override this.
